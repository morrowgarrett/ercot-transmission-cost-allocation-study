# Limitations

- v1 is not customer-level settlement reconstruction.
- TDSP and proxy-class views are analytical overlays, not tariff-bill truth.
- CBTCA operational burden may rely on imperfect congestion proxies if direct congestion components are unavailable.
- LMP minus system lambda is a congestion-plus-loss proxy, not a pure congestion signal.
- CBTCA planning burden in this public framework still depends on constraint scoring inputs that may be assembled from simplified public-safe structures rather than full market-system reconstructions.
- Net-load variants may depend on estimated BTM offsets if direct observability is weak.
- Synthetic comparison runs validate interface consistency and metric behavior, not substantive ERCOT policy conclusions.
- Gaming-exposure metrics in the scorecard are currently heuristic and should be treated as challenge targets, not settled fact.
- Public repo artifacts must not include licensed vendor data.
